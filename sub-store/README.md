# Sub-Store 节点筛选脚本

`operators/keep-core-premium-and-remote.js` 用于生成一个适合当前 OpenClash 分组的精简节点订阅。

`operators/filter-sort-by-region-and-quality.js` 是独立的 Script Operator：删除订阅说明项、规范重复节点名称，并按地区排序；同一地区内优质节点排在普通节点之前。

它保留以下节点：

- 香港、台湾、美国、日本、新加坡中符合优质条件的节点；
- 其他所有国家/地区的节点（包括冷门国家）；
- 名称无法可靠识别国家的节点，采用保守保留，避免误删供应商的特殊命名。

它会移除五个核心国家中不符合优质条件的普通节点。因此 `美国.T1 | 直连`、`香港.R1 | 直连` 这类没有优质标签的节点不会出现在输出中。

优质标签与现有配置保持一致：`进阶IEPL`、`专线`、`原生`、`家宽`、`星链`、独立的 `2x`、`3x`、`4x`、`5x`。普通 `IEPL`、`CN2`、`IPLC`、`进阶`、`1x` 不算优质；`0.5x`、`1.5x`、`15x` 也不会误判为 `5x`。

## 使用方式

1. 在 Sub-Store 建一个合集订阅，把机场订阅和私有节点源都加入其中。
2. 对这个合集添加 **脚本过滤（Script Filter）**，将 `operators/keep-core-premium-and-remote.js` 的完整内容粘贴进去（或使用该文件可访问的脚本 URL）。这是推荐方式。
3. 预览输出：五国普通节点应消失，五国优质节点与全部其他国家节点应保留。
4. 将该 Sub-Store 输出订阅地址填入客户端或 OpenClash。`meta.ini` 仍负责策略组和分流规则；本脚本只负责节点筛选。

该文件也兼容 **脚本操作（Script Operator）**，但不要把它粘进“正则过滤”或“正则删除”。若曾使用旧版本脚本并看到“保留节点 0”，请删除旧操作后重新添加一个“脚本过滤”，再粘贴本文件的完整新内容。

不要在 Sub-Store 再添加全局 `0.1x`、`福利` 等正则删除规则；这会改变目前约定的筛选范围。
